## 🔍 Root Cause Analysis

### Why IDE Shows False Positives

1. **Lombok Annotation Processing**
   - IDE doesn't process `@Setter` and `@Getter` correctly
   - Methods are generated at compile time, not visible in source
   - IDE shows errors for methods that actually exist

2. **JPA Enum Mapping**
   - `@Enumerated(EnumType.STRING)` confuses IDEs
   - IDE thinks field is String type, but it's actually `DiscountType` enum
   - Annotation only affects database storage, not Java type

3. **IDE Cache Issues**
   - Stale cache showing old errors
   - Incomplete indexing
   - Background indexing not finished
