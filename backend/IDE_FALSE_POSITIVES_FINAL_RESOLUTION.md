## Why IDE Shows False Positives

### Root Cause 1: Lombok Annotation Processing

The IDE doesn't process `@Setter` and `@Getter` annotations correctly, so it can't see the generated methods.

**What Actually Happens:**
- Lombok generates `setDiscountType(DiscountType)` at compile time
- IDE shows error: "setDiscountType(String) not applicable for DiscountType"
- IDE incorrectly thinks the method takes String

**Reality:**
- Method signature: `public void setDiscountType(DiscountType)`
- Generated by Lombok `@Setter` annotation
- Verified in bytecode above

### Root Cause 2: JPA Enum Mapping

`@Enumerated(EnumType.STRING)` confuses the IDE.

**What Actually Happens:**
- Annotation tells JPA to store enum as String in database
- Does NOT change Java type - still `DiscountType` enum
- IDE thinks field is String type

**Reality:**
- Field type: `DiscountType` enum
- Database storage: String (via JPA)
- Java type: DiscountType enum
- Verified in bytecode above

### Root Cause 3: IDE Cache Issues

IDE cache is stale and showing old errors.

**Solution:**
- Clear IDE cache
- Rebuild project
- Enable annotation processing
